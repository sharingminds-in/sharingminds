Project Architecture

Young Minds Mentor-Mentee Platform

A high-level architecture view of the current Next.js application: role-aware dashboards, tRPC routers, REST route handlers, domain services, Drizzle/PostgreSQL persistence, storage, authentication, payments, messaging, LiveKit, and AI integrations.

UI Layer Next.js App Router pages, dashboard shell, role-specific components, and providers.
API Boundary tRPC for typed business operations plus REST handlers for uploads and webhooks.
Domain Services Server-side business logic for admin, mentor, booking, content, payment, and messaging flows.
Persistence & External Drizzle/PostgreSQL, object storage, BetterAuth, Razorpay, LiveKit, email, and AI services.

System Architecture

End-to-end structure from browser roles to API routes, services, persistence, and integrations.

Layer 1

Users and Browser Entry

Three role groups use the same web application. Role-aware routing and dashboard sections decide which experience each user sees.

Admin Manages mentors, users, content review, policies, enquiries, and operational controls.
Mentor Manages profile, availability, sessions, content, courses, and mentee interactions.
Mentee Discovers mentors, books sessions, manages profile, learns from content, and messages mentors.
Web Browser Loads the Next.js app, authenticates with BetterAuth, and calls tRPC or REST endpoints.
↓

Layer 2

Next.js App Router UI

The frontend is composed from App Router pages, a shared dashboard shell, feature modules, and global providers.

app/* pages Landing, dashboard, mentor signup, learning, meetings, sessions, recordings, and review flows.
components/dashboard Role-aware dashboard shell and section renderer for admin, mentor, and mentee experiences.
components/* Feature UI for admin, mentor, mentee, booking, messaging, content, notifications, and shared profile screens.
providers + contexts Auth, theme, query client, app state, and cross-cutting client context.
↓

Layer 3

Client Data Access

Most business operations go through typed tRPC hooks. Multipart uploads and third-party webhooks use direct route handlers.

hooks/queries/* TanStack Query wrappers for caching, invalidation, loading states, and mutations.
lib/trpc/react Typed tRPC client used by dashboard and feature components.
Direct fetch Used for multipart file flows such as uploads, mentor applications, admin mentor creation, and legacy route handlers.
↓

Layer 4

Server API Boundary

API requests enter through a small set of route boundaries. tRPC owns typed app operations, while REST handlers cover uploads, auth, webhooks, and streaming.

app/api/trpc/[trpc] Single tRPC endpoint that dispatches to all application routers.
app/api/auth/[...better-auth] BetterAuth session, email/password, Google OAuth, and LinkedIn OAuth handlers.
app/api/* routes Admin mentor creation, mentor application uploads, content uploads, LiveKit webhooks, Razorpay webhooks, recordings, locations, and SSE.
↓

Layer 5

tRPC Routers

Routers group the public API surface by domain. They validate input, enforce procedure-level access, and delegate business logic to services.

Core and role workflows

  • auth, public, profile, analytics
  • admin, mentor, bookings
  • content, learning

Platform features

  • messaging, notifications
  • payments, subscriptions
  • recordings, chatbot
↓

Layer 6

Server Domain Services

Services contain the actual business rules and orchestration. UI and route handlers should stay thin and call these modules.

lib/admin Admin consoles, mentor verification, user provisioning, policy management, and audit actions.
lib/mentor + lib/profile Mentor applications, profile updates, availability, lifecycle state, and profile mapping.
lib/bookings + lib/meetings Session booking, access policy checks, scheduling rules, LiveKit meeting access, and session lifecycle.
lib/content + courses + learning Mentor content, courses, enrollments, feedback, progress, and learning analytics.
lib/messaging + notifications Threads, permissions, SSE messaging, notifications, and delivery records.
lib/payments + subscriptions Razorpay checkout, subscription plans, limits, payment state, and billing workflows.
lib/recordings + livekit Recording lifecycle, playback URL generation, room events, and LiveKit integration.
lib/chatbot AI chatbot flows, message storage, and AI-powered insight features.
↓

Layer 7

Persistence and Storage

Drizzle owns relational persistence. Object storage handles files, profile images, resumes, content files, and recordings.

lib/db Drizzle client, schema exports, migrations, and database helpers.
PostgreSQL Users, roles, mentors, mentees, content, bookings, sessions, payments, notifications, audit trails, and availability.
lib/storage Storage abstraction for Supabase Storage or S3, signed URLs, uploads, and deletes.
↓

Layer 8

External Integrations and Cross-Cutting Rules

External systems and shared enforcement modules sit around the domain services and route boundaries.

External services

  • Google OAuth and LinkedIn OAuth via BetterAuth
  • Razorpay checkout and webhooks
  • LiveKit rooms, tokens, webhooks, and recordings
  • Nodemailer SMTP for email notifications
  • Google AI SDK / Gemini for chatbot features

Cross-cutting modules

  • lib/api/guards for admin, mentor, and mentee role checks
  • lib/access-policy for runtime access policy enforcement
  • lib/audit and DB audit tables for operational traceability
  • zod schemas and lib/validations for input contracts

Primary Runtime Flow

Common path for dashboard interactions and feature mutations. 

sequenceDiagram
  participant User as Browser User
  participant UI as Next.js UI
  participant Hook as TanStack Query Hook
  participant TRPC as tRPC Endpoint
  participant Service as Domain Service
  participant DB as Drizzle + PostgreSQL
  participant Ext as External Service

  User->>UI: Interact with dashboard or public page
  UI->>Hook: Query or mutation
  Hook->>TRPC: Call router procedure
  TRPC->>Service: Validate input and execute use case
  Service->>DB: Read or write application data
  Service-->>Ext: Optional email, payment, LiveKit, AI, or storage call
  DB-->>Service: Persisted result
  Service-->>TRPC: Typed response
  TRPC-->>Hook: JSON response
  Hook-->>UI: Cached UI state update
  UI-->>User: Updated screen

System Boundary Architecture

Internal platform services, persistence, and external providers shown as a readable system-level map.

Internal Application Boundary

Next.js Web Application App Router pages, role dashboards, feature components, and client providers.
tRPC API Layer Typed procedure boundary for admin, mentor, mentee, booking, content, payment, and messaging operations.
REST Route Handlers Multipart uploads, mentor applications, admin mentor creation, webhooks, recordings, locations, and SSE.
Domain Services Business rules in lib/admin, lib/mentor, lib/bookings, lib/content, lib/payments, lib/messaging, and related modules.
Authorization Guards Role checks, session validation, access policies, subscription limits, and admin-only protections.
Audit and Notifications Admin audit trails, mentor profile audit, session audit, notification records, and email event tracking.
Internal services read/write data and call providers

Internal Persistence Boundary

PostgreSQL Database Primary relational store for users, roles, mentors, mentees, sessions, bookings, content, payments, and audits.
Drizzle ORM Schema modules, typed queries, migrations, and server-side database access.
Object Storage Supabase Storage or S3 abstraction for profile images, resumes, mentor content files, and recordings.
Auth Tables BetterAuth-compatible users, sessions, accounts, and verification records stored through Drizzle.
External integrations

External Service Boundary

Google OAuth Social authentication provider configured through BetterAuth.
LinkedIn OAuth Additional social authentication provider for account login flows.
Razorpay Session checkout, payment processing, and webhook-driven payment status updates.
LiveKit Video rooms, access tokens, room events, recording webhooks, and session playback integration.
Email Provider Nodemailer SMTP delivery for mentor application, verification, coupon, and notification emails.
Google AI / Gemini AI chatbot flows, mentor discovery assistance, and AI-backed insight features.
Supabase Storage or S3 External object storage provider behind the internal lib/storage abstraction.
