5. TOKEN ECONOMICS (VERY IMPORTANT)
The infrastructure must follow: HIGH SIGNAL / LOW TOKEN design.
The objective is NOT: minimum tokens.
The objective is: maximum trust per token.

6. REALISTIC TOKEN ECONOMICS
USER INPUT RANGE
Typical: 40–120 tokens
Examples:
⦁	“I feel stuck in my career.”
⦁	“I want to study abroad but I’m confused.”
⦁	“I don’t know if I should switch industries.”

AI OUTPUT RANGE:
LIGHT MODE
Target: 80–180 tokens
Use case:
⦁	homepage visitors
⦁	casual users
⦁	low-intent discovery

STANDARD MODE
Target: 250–550 total conversation tokens
Use case:
⦁	most active users
⦁	strategic clarity conversations
⦁	career or business uncertainty

DEEP CLARITY MODE
Target: 700–1200 total conversation tokens
Use ONLY when:
⦁	user engagement is high
⦁	emotional seriousness is clear
⦁	conversion probability is high
⦁	user repeatedly interacts
Deep mode is commercially acceptable because:
⦁	AI cost remains extremely low relative to session revenue
⦁	deeper trust significantly improves conversion probability
